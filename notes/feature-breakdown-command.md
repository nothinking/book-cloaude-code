# 기능 분해 커스텀 커맨드 (Feature Breakdown Custom Command)

구현하고 싶은 기능을 입력하면, 체계적으로 작은 단위로 분해된 실행 계획을 제공받을 수 있습니다.

## 사용법

```
/feature-breakdown <구현하고 싶은 기능>
```

## 프롬프트

다음 기능을 체계적으로 분해하여 실행 가능한 작업 단위로 나누어 주세요:

**구현하고 싶은 기능:** $ARGUMENT

다음 구조로 분석해주세요:

### 1. 기능 분석

- **핵심 목표**: 이 기능이 해결하려는 문제와 기대 결과
- **범위 정의**: 포함되는 것과 제외되는 것
- **사용자 시나리오**: 주요 사용 케이스 2-3개

### 2. 아키텍처 분해

- **프런트엔드 구성요소**: UI 컴포넌트, 페이지, 상태 관리
- **백엔드 구성요소**: API 엔드포인트, 데이터베이스, 비즈니스 로직
- **외부 연동**: 필요한 외부 서비스나 라이브러리

### 3. 작업 분해

각 작업을 다음 형식으로 나열:

- **[우선순위] 작업명**
  - 설명: 구체적인 작업 내용
  - 예상 시간: 1-4시간 단위
  - 의존성: 선행 작업이나 의존관계 명시
  - 테크 위험: 크리티컬 요소나 어려운 상황에 주의 필요

### 4. 체크포인트

- **병합 지점 가능**: 중지 후 전환할 수 있는 작업 지점
- **블로킹 포인트**: 다른 팀원이나 의사결정자에 대한 확인 필요
- **스파이크 필요**: 반드시 사전에 파악해 두어야 하는 의견/기술
- **불확실성 필요**: 리스크가 높거나 먼저 해결해야 할 작업들

### 5. 검증

- **사용성 테스트**: 사용자 관점에서의 검증 포인트
- **기술 검증**: API 연동성이나 UI 연동성에 대한 요소 검증
- **파일럿 적용**: 작업 항목 생성 후 파일럿 적용 가능성
- **마일스톤**: 주요 중간 목표 지점

### 6. 진행 추적

- 작업 항목화: 1-4시간 단위의 작은 크기로 분류
- 우선순위: 비즈니스 가치나 리스크 고려
- 테스트 계획: 각 작업마다 완료 조건을 명확히 지정

## 결과물

위 분석 내용을 정리하여 필요한 곳에 구조화된 정보로 제공합니다.
